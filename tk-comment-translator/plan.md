# tk-comment-translator — Implementation Plan

## Overview

`tk-comment-translator` will be a standalone microservice package that translates TikTok comments into a bilingual JSON output, following the same general approach as `tk-srt-translate`:

`Parse input -> Preprocess -> Batch translate -> Validate -> Write bilingual output`

The package should remain independent from the orchestrator and other services. Like the existing tools in this repo, it should expose a CLI entry point and operate on files/stdout rather than requiring direct imports from other packages.

---

## Goal

Accept comment JSON produced by `tk-comment-checker` and return translated comment JSON where each item preserves the original metadata and adds a translated text field.

### Expected input

```json
[
  {
    "user": "Max",
    "username": "maximevaillancourt",
    "text": "C est bien de voir de jeunes artistes...",
    "likes": 6
  }
]
```

### Expected output

```json
[
  {
    "user": "Max",
    "username": "maximevaillancourt",
    "text": "C est bien de voir de jeunes artistes...",
    "translation": "看到年轻艺术家不吸毒而且写出有思想的歌词，这真的很好。",
    "likes": 6
  }
]
```

## Planned CLI

Keep the package shape aligned with the repo's other standalone tools.

```bash
tk-comment-translator <input.json> [--output FILE] [--model MODEL] [--batch-size N] [--max-tokens N] [--temperature N] [-v]
```

Planned behavior:

- Read comments from a JSON file generated by `tk-comment-checker`
- Skip empty comments instead of sending them to the model
- Preserve comment order in output
- Write translated JSON to a file or stdout

---

## Planned Pipeline

### 1. Parse input

- Load JSON array
- Validate each item has the expected fields: `user`, `username`, `text`, `likes`
- Assign an internal sequential `id` for translation tracking

### 2. Preprocess

- Filter comments with empty or whitespace-only `text`
- Trim whitespace
- Preserve original comment metadata for later merge
- Convert working items into a compact translation payload such as:

```json
[
  {"id": 1, "text": "C est bien de voir de jeunes artistes..."}
]
```

### 3. Batch translate

- Reuse the same batching strategy as `tk-srt-translate`
- Translate comments in small batches
- Use deterministic generation by default
- If a batch fails validation, split and retry recursively

### 4. Validate

- Response must be valid JSON
- Output count must match batch input count
- Output ids must match in order
- Each translated string must be non-empty unless the source is intentionally skipped

### 5. Merge output

- Merge translations back into the original comment objects
- Add a new field such as `translation`
- Preserve original ordering and metadata

## Design Decisions

- Keep this as a top-level standalone package, not a submodule of `tk-comment-checker`
- Do not merge this package into `tk-srt-translate`; keep separate CLIs and package boundaries because the input/output contracts are different
- Mirror the LLM calling and batch fallback logic from `tk-srt-translate`
- Keep output JSON-first, since comment data is already JSON-native
- Do not mutate or overwrite the source text; add translated text alongside it

### Shared Logic Guidance

This package should stay separate from `tk-srt-translate`, but it should be written so the common translation mechanics can be shared or copied cleanly.

Important constraint:

- The assigned coder should assume they cannot read or inspect the `tk-srt-translate` source code
- This plan must therefore be treated as the complete implementation brief
- Do not rely on "check the SRT package for details" during implementation

Common pieces that should follow the same design as `tk-srt-translate`:

- LLM invocation wrapper
- Batch processing loop
- Recursive split-and-retry behavior
- JSON extraction / parse recovery from raw model output
- Response validation patterns based on ordered `id` matching
- Translation config fields such as `model_path`, `batch_size`, `max_tokens`, `temperature`, `max_retries`, `retry_delay`

Pieces that must remain specific to `tk-comment-translator`:

- Comment JSON parsing
- Empty-comment filtering rules
- Comment translation prompt
- Comment-specific validation details
- Output merge shape with original metadata and new translation field

The implementation should prefer small, isolated functions so a later refactor can extract a shared internal translation core without changing CLI behavior.

## Core Logic Snippets

These snippets are intentionally implementation-oriented so a junior coder can build the service from this document alone. They are examples of the expected control flow, not a requirement to match exact syntax.

### 1. Main translation flow

```python
from pathlib import Path
import json


def translate_comments_file(input_path: Path, output_path: Path, config) -> Path:
    raw_items = json.loads(input_path.read_text(encoding="utf-8"))

    comments = parse_comments(raw_items)
    prepared_comments, translation_items = preprocess_comments(comments)
    translations = translate_all_batches(translation_items, config)
    merged = merge_translations(prepared_comments, translations)

    output_path.write_text(
        json.dumps(merged, ensure_ascii=False, indent=2),
        encoding="utf-8",
    )
    return output_path
```

### 2. Parse and preprocess

```python
def parse_comments(raw_items: list[dict]) -> list[dict]:
    if not isinstance(raw_items, list):
        raise ValueError("Input must be a JSON array")

    parsed = []
    for index, item in enumerate(raw_items, start=1):
        if not isinstance(item, dict):
            raise ValueError(f"Comment #{index} is not an object")

        for field in ("user", "username", "text", "likes"):
            if field not in item:
                raise ValueError(f"Comment #{index} missing field: {field}")

        parsed.append(
            {
                "id": index,
                "user": item["user"],
                "username": item["username"],
                "text": str(item["text"]),
                "likes": item["likes"],
            }
        )
    return parsed


def preprocess_comments(comments: list[dict]) -> tuple[list[dict], list[dict]]:
    translation_items = []

    for comment in comments:
        comment["text"] = comment["text"].strip()
        if comment["text"]:
            translation_items.append(
                {
                    "id": comment["id"],
                    "text": comment["text"],
                }
            )

    return comments, translation_items
```

### 3. Batch translation with split-and-retry

```python
def translate_all_batches(items: list[dict], config) -> dict[int, str]:
    results = {}

    for start in range(0, len(items), config.batch_size):
        batch = items[start:start + config.batch_size]
        batch_result = translate_batch_with_retry(batch, config)
        results.update(batch_result)

    return results


def translate_batch_with_retry(batch: list[dict], config, depth: int = 0) -> dict[int, str]:
    if not batch:
        return {}

    last_error = None

    for _ in range(config.max_retries + 1):
        try:
            prompt = build_prompt(batch)
            raw_response = call_llm(prompt, config)
            rows = parse_model_json(raw_response)
            validate_translation_rows(rows, batch)
            return {row["id"]: row["translation"] for row in rows}
        except Exception as exc:
            last_error = exc

    if len(batch) == 1:
        raise RuntimeError(f"Single comment translation failed for id={batch[0]['id']}: {last_error}")

    mid = len(batch) // 2
    left = translate_batch_with_retry(batch[:mid], config, depth + 1)
    right = translate_batch_with_retry(batch[mid:], config, depth + 1)
    return left | right
```

### 4. Model response parsing and validation

```python
import json
import re


def parse_model_json(raw_response: str) -> list[dict]:
    text = raw_response.strip()

    try:
        return json.loads(text)
    except json.JSONDecodeError:
        match = re.search(r"\[.*\]", text, re.DOTALL)
        if not match:
            raise ValueError("Model output does not contain a JSON array")
        return json.loads(match.group(0))


def validate_translation_rows(rows: list[dict], batch: list[dict]) -> None:
    if not isinstance(rows, list):
        raise ValueError("Model output must be a JSON array")

    if len(rows) != len(batch):
        raise ValueError("Output count does not match input count")

    for row, source in zip(rows, batch):
        if not isinstance(row, dict):
            raise ValueError("Each output item must be an object")

        if row.get("id") != source["id"]:
            raise ValueError("Output ids must match input ids in order")

        translation = row.get("translation")
        if not isinstance(translation, str) or not translation.strip():
            raise ValueError(f"Missing translation text for id={source['id']}")
```

### 5. Merge translations into original comments

```python
def merge_translations(comments: list[dict], translations: dict[int, str]) -> list[dict]:
    merged = []

    for comment in comments:
        item = {
            "user": comment["user"],
            "username": comment["username"],
            "text": comment["text"],
            "likes": comment["likes"],
        }

        if comment["id"] in translations:
            item["translation"] = translations[comment["id"]]
        else:
            item["translation"] = ""

        merged.append(item)

    return merged
```

### 6. Prompt shape

The model should be asked to return JSON only, with one output row per input row.

```python
def build_prompt(batch: list[dict]) -> str:
    payload = json.dumps(batch, ensure_ascii=False)
    return f"""
You are a professional French-to-Chinese comment translator.

Return JSON only.
Output format:
[{{"id": 1, "translation": "..."}}]

Rules:
1. Keep every id exactly as given
2. Keep the same order as input
3. Do not skip any item
4. Do not add explanation text

Comments to translate:
{payload}
""".strip()
```
